# E2 時區跨日誤報——UTC 帳本 vs 本地 ended 兩時間源混用

## 現場(2026-07-20 00:30 台北)
Kotlin 檢核補全的例行全套測試紅一條 `t_supersede_triggers_cascade`(M4 E2E)——與該次改動零關聯。stash 驗證乾淨 HEAD 同紅;昨日兩次全套皆綠 → date-rollover 觸發。烟槍=同一次 supersede 輸出:`CASCADE c-20260719162929-…`(UTC)與 `ended=2026-07-20`(本地)並排。

## 病因
`scripts/lumos` E2 ledger 抑制:`_lts[:10] >= str(ended)[:10]`——左邊 UTC iso、右邊本地日期。台北凌晨(UTC+8 的 00:00-08:00)左邊日期字串還在昨天 → 「已判過」被誤判「判定早於翻案」→ 重報。

## 修法
比較點轉本地日期(寫入側 UTC 不動,歷史帳相容);回歸測試手造本地午夜的 UTC 表示,任何時區/時辰確定性。

## 殘餘
- `rel_cascade_create` 的 cascade-id 鑄造 ts 仍為 UTC(:4980)——純識別符不進任何比較,不改;若日後有人拿 id 內時戳做日期邏輯,本 Issue 為前車之鑑。
- 全庫其他「日期字串比較」點未逐一盤(E2 是唯一混源比較;`rel-cascade list --stale` 為 UTC vs UTC 同源無虞)——若再冒同族,先查兩端時間源。
