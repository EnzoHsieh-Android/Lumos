# 檢索優化_調研

## 目標與方法

補「中段」：檢索排序（search 現為字母序零排序）、相關節點推薦（現為無權重 BFS）、主動提醒降噪（impact hook 實戰一次灌 50+ 節點）。deep-research 五路 → 24+ 來源 → 逐條 3 票對抗查證 → 8 組存活（3 條被殺）。全部裁定 **borrow-design**（stdlib 原生實作）。

PRIOR-ART: 本節點即先問世界存查——BM25F/Whoosh/HippoRAG/LEGO-GraphRAG/Borodin 小圖實證/Obsidian Graph Analysis/Mozilla frecency 七支文獻線真搜真驗。

## 存活結論（機制 × 借法）

### 1. 查詢排序：BM25F（高信度 3-0，原作者論文）
- **正確做法**：欄位權重乘在 term frequency 上、在飽和函數**之前**（等價「標題重複 w 次」），單一虛擬文件跑一次 BM25。**最容易踩的坑**：分欄位各算 BM25 再線性相加——會破壞非線性飽和、降效（Robertson CIKM 2004 原文警告；Elasticsearch/Sourcegraph Zoekt 產品實作佐證）。
- **公式直接抄 Whoosh**（原始碼逐行驗證、純 stdlib math）：`score = idf * tf*(K1+1) / (tf + K1*((1-B) + B*fl/avgfl))`，B=0.75、K1=1.2；**IDF 用平滑式 `log(N/(df+1))+1`**——數值驗證過小語料下全文都出現的高頻詞仍為正分（Robertson 原式會給負分，小 vault 必炸）。

### 2. 相關節點推薦：BFS 距離衰減優先、PPR 備選（高信度但皆為外推）
- 文獻主流是 **Personalized PageRank**（HippoRAG NeurIPS 2024 勝 SOTA RAG 20%、HippoRAG 2 保留、fast-graphrag 產品化、LEGO-GraphRAG 實證 PPR 全面勝 random walk）——但實證全在大 KG。
- **小圖反直覺實證（Borodin ACM TOIT 2005，對本 vault 規模最相關）**：千級節點圖上 in-degree 排序(61%)已勝 PageRank(48%)/HITS(47%)；**全場最佳是 BFS 距離衰減變體(78%)**：節點貢獻 = 1/2^k（k=最短距離、每節點只計一次）——**正是現有無權重 BFS 的最小升級**，且按可達性非路徑數計分、天然抗密集叢集劫持（vault 的 MOC/高互鏈群不會吃掉推薦）。
- 輔助訊號（Obsidian Graph Analysis，原始碼驗證）：**共引**（兩節點被同一節點共同引用，同句加權=2nd-order backlinks）與**Jaccard 鄰域相似**（|共同鄰居|/|聯集|,純集合運算）。⚠ 該 plugin 的 Adamic-Adar 說法被查證否決,要用需另驗原始碼。
- **裁定**：v1 用 BFS-decay(1/2^k)+共引+Jaccard 的加權組合（改動最小、小圖實證最強）；PPR 留 v2 選項（大 vault 或 A/B 後）。

### 3. 融合鐵律（LEGO-GraphRAG PVLDB 2025，3-0）
純結構訊號只是 baseline（實測墊底）——**圖分數必須與詞彙相關性（BM25）融合**，不可單獨用。⚠ 融合公式（線性加權 vs RRF）無直接證據,開放題。

### 4. 排序的時間軸：Mozilla frecency（高信度，現行 Firefox 文件）
`score = Σ weight_visit × e^(-λΔt)`，λ=ln(2)/30（30 天半衰期，stdlib math.exp）；事件依意圖強度分權重桶（手動查 > 自動注入）；取樣最近 N 次控成本。lumos 對應：節點被 `context`/`search` 手動讀 = 高權重事件、hook 自動注入 = 低權重——**前提是要開始記「節點被誰讀過」**（現在沒有這個帳）。⚠ legacy 五桶硬編權重數值被否決,借新版指數衰減。

## 未產出結論的調查點（誠實缺口）

- **中文無空格分詞**（字元 bigram vs 逐次掃描）：零存活 claim → 自己 A/B（小語料逐次掃描+bigram 可能夠用,未驗證）。
- **MMR 去冗餘**與 top-k 閾值實踐：零存活 → 拿 impact hook 真輸出經驗調參。
- 所有圖演算法在「數百節點中文 vault」皆無直接實證——落地必配真 vault 評測（人工標註相關對）。

## 候選落地優先序（待裁定）

1. search 排序：BM25F(Whoosh 公式+平滑 IDF+欄位 tf 加權) + 中文 bigram 自行 A/B
2. impact/related 升級：BFS-decay(1/2^k)+共引+Jaccard 加權,與 BM25 融合(公式自定+真 vault 評測)
3. hook 降噪：分數排序 top-k(5-10)+閾值,MMR 經驗調參
4. frecency：先開「節點讀取事件」帳,再上指數衰減排序(依賴 1-3 的分數框架)
