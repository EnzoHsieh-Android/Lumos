# 業務簽核留痕_計劃

## 目標

解調研裁定④：`[test:]`/`[audit:]`/`kill` 都是 verification（技術上對）；validation（業務上還對不對）只有人能判——工具能做的是把「人點頭」變成可查的留痕。

PRIOR-ART: ① 最小解=一個留痕指令+一支 gov load，仿 code-loop pass ledger。② 世界解過：sign-off/approval ledger 是審計實務標配；本 repo 的 anchor approve/code-loop pass 已是同型先例。③ 裁定=borrow-design（自家 ledger 樣板複用）。

## 變更規格

- **指令** `lumos signoff <節點> --note "<確認了什麼>" [--by <人>]`：append `docs/.signoff-log.jsonl`（{ts,node,by,note,commit}）+ 節點 frontmatter 純量 `signed_off: <YYYY-MM-DD>`（T1 set 白名單加 `signed_off`）。語意=「業務面在此日期被此人確認過」；何時該簽由人/流程判，工具不擋。
- **gov 第 6 支 load**（gate="signoff"）+ 老三處硬編同步（scaffold gitignore、cochange 排除——cochange Gate CC 會提醒漏改的夥伴,本次是它的實戰）。
- `lumos context` 讀節點時 signed_off 自然顯示（frontmatter 純量,免改）。

## 測試計畫

- signoff 寫後自驗（ledger 一筆+frontmatter 純量）；gov 撈得到；缺 --note 拒絕；重簽覆蓋日期、ledger 累加。
