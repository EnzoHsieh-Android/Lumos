# 主動影響幅度偵測 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.
> **設計權威**:[[主動影響幅度偵測_計劃]](§1 架構 / §2 lumos impact 原語 / §3 hook / §4 天花板 / §5 測試 / 審計修正紀錄)。本計畫每 task 的算法細節以設計 § 為準,不重述,只給可執行測試 + 實作要點 + 驗證指令。

**Goal:** 實作 `lumos impact` 原語 + PreToolUse hook,動手前主動算出將被改動 code 的 downstream 影響節點集、推到 Claude 眼前(lumos 關聯節點自動修復的偵測地基)。

**Architecture:** `lumos impact --file <code> --json --repo <root>` 純記憶體即時算「直接(body inline-code 反查)+ 間接(env.edges BFS)」影響清單、出機器可讀 `--json`;PreToolUse hook 攔 Edit/Write/MultiEdit → 過濾 code → TTL 節流 → 呼叫原語 → `additionalContext` 注入(不擋)。lumos 只算、Claude 只判、hook 只搬運節流。

**Tech Stack:** python3 標準庫(零依賴,同 scripts/lumos);Claude Code hooks(PreToolUse、hookSpecificOutput.additionalContext);既有 lumos 原語 `load_vault`/`_refcheck_scan`/`env.edges`/`link_target`/`env.resolve`/`extract_contracts`/`IRREVERSIBLE_RE`/`as_list`。

## Global Constraints

- **零第三方依賴**:純 python3 標準庫(scripts/lumos 慣例)。
- **vault-free / fail-open**:任何錯誤絕不擋 Claude;rc 協定 `0`=成功 / `3`=vault 找不到(hook 印 debug) / 其他非 0=內部錯(hook 純靜默)。
- **不改 `env.edges`/`load_vault`/`Note` 建構**(全 lumos 共用核心 infra);impact 的反查靠**呼叫時重讀盤**餵 `_refcheck_scan`,via 二次反查只在 impact BFS 內做。
- **code→node 反查只認 body inline-code**(`core_refs` 是跨 repo 節點指針非 code 路徑,不掃)。
- **路徑一律 repo-relative**(hook 傳絕對路徑 → `os.path.relpath(abs, repo_root)` 再比對)。
- **`--json` 是機器可讀契約**(下游自動修復消費):欄位型別/可空/枚舉/presence 依設計 §2 schema 定死。
- **測試進 `scripts/test_lumos.py`**(既有測試框架,現 500+ passed);每 task 綠燈才進下一 task。
- **hook 不進 anchor**(比照現有 `scripts/hooks/claude/*.py` 現況)。
