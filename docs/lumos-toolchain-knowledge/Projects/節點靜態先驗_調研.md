# 節點靜態先驗_調研（使用者提案「三維係數」）

使用者提案：給每個節點一個「三維係數」讓搜尋/巡檢/分類更精確。照家規先問世界：網搜評審（真搜附源）× Codex/GPT-5.6 架構評審，雙方獨立作答、結論收斂。

PRIOR-ART: ① 最小解=在已正規化的融合分 R 上做 bounded 調整（不動候選集、不動固定席）② 世界解過（IR/ES/Firefox 三條成熟線，見下）③ 裁定=borrow-design（Obsidian 社群無現成輪子——maturity 分級沒人拿進排序，Omnisearch recency 加權停在 feature request）。

## 雙評審收斂結論

**值得做，但不是「每節點手填三個數字」——是三個機械推導、可消融的 ranking features。**

| 維度 | 優先序 | 世界做法 | 裁定 |
|---|---|---|---|
| **型別先驗（MOC 降權）** | 高（有實證） | 企業搜尋標準處置：hub/index 頁乘法降權或隔離（GOV.UK boost/demote、ES function_score multiply、Google Vertex boost/bury）；HITS 理論：MOC 是 hub 非 authority，hub 該降 | **第一個可合併版**：`type=moc` 乘 0.3–0.5；goldset 已實證 MOC=關鍵字磁鐵（16 爭議例全判噪音）。明確搜索引頁的導航 query 應取消降權 |
| **新鮮度 frecency** | 中（先建事件帳） | Firefox frecency：事件加權×指數衰減；重寫版教訓=別存時變分數（bug 458801「high-maintenance」）、係數越少越好；學術：freshness 只幫時間敏感 query | **Codex 關鍵反對**：拿 `updated` 冒充使用事件會**獎勵頻繁改稿、懲罰穩定合約**——需真實「被採用」事件帳（context/選取,opt-in shadow ledger）才進正式分。日期衰減僅輔助、半衰期 90–180 天、查詢時現算不落地 |
| **權威度 in-degree** | 低（shadow 驗證） | Najork 大規模實證：HITS/PageRank ≈ in-degree;Craswell SIGIR'05：多個圖特徵冗餘（加 PageRank 後 indegree 權重歸零）→ 只留一個;TREC：連結先驗只幫導航型查詢（lumos search 恰是）;內網文獻：策展 wikilink=無 spam 乾淨背書 | `sat(backlink 數)`,**剔除來自 MOC 的出鏈**（hub 灌水）;99–235 節點跑 PageRank=殺雞用牛刀且邊語意混雜（related/plan_refs/verified_by 非同質背書）;對 held-out 無穩定增益就刪 |

## 併分公式（世界標準答案）

```
final = type_mult(node) × [ R + Σ wᵢ·sat(featureᵢ) ]
  R          = 0.6·BM25F + 0.4·G(現制不動;候選集/固定席不動)
  type_mult  : moc 0.3–0.5;superseded/stale 0.3;其餘 1.0(降權用乘法——比例效果跨 query 一致)
  sat(x)     = x/(x+pivot),pivot=全庫幾何平均(ES rank_feature 零調參預設);加分用飽和加法,嚴禁裸加裸乘
  紀律       : 靜態貢獻合計 ≤ 總分 20–30%(Craswell 最優 w 即此量級);權重收單一 ranking config,嚴禁散落 frontmatter
```
- 學術金標：Craswell/Robertson SIGIR 2005（log 域 sigmoid 併入 BM25,.GOV MAP .458→.523）。
- 加法為何安全：本架構 R 已正規化 [0,1]（ES 說「裸加固定值跨 query 不穩」是對未正規化 BM25 而言）。

## 消融階梯（goldset 當裁判,Codex 版）

A0 現況 → A1 只加 MOC 乘法降權 → A2 +事件式 frecency → A3 +sat(typed-backlink,剔 MOC 邊) → A4 全部。權重只在 train split 調一次、凍結後跑 held-out;裁決指標=search nDCG@5/held 不倒退/hook P@8/過閾數/固定席噪音/安全固定席(合約/事故類)recall=100%(標2必看的 in-output recall 另報、無機保)。**只有 A1 夠格直接合併**;A2 無事件帳不進正式分;A3 held-out 無增益即刪。

## 對 hook 面三病因的誠實界定（Codex）

靜態先驗**只是局部幫助,不是主修復**：①閾過鬆的根因是 direct 基底 0.5 恆過動態閾（max ≤0.25）——要 direct 專屬閾/名額或最低詞彙證據;②trigger 準度與先驗無關（incident=1.0 pinned,壓它反破壞安全席）——逐條回放 9/44 噪音修 pattern;③direct 席過寬要收窄反查/拆 exact-reference vs 弱路徑。**hook v1.1 順序：閾值與 direct 規則 → trigger precision → 最後才 static prior**。

## 手標 vs 機械推導（裁定）

特徵機械推導、語意由人標：人只負責既有 type/status/合約/邊;嚴禁手填 `authority: 0.83` 類浮點（GIGO/漂移/自我強化）;若需例外只開離散可 lint 的語意枚舉（如 `ranking_role: index|canonical|historical`）,不開任意數字逃生門。

## A3/A1.5 消融結果（2026-07-11 深夜,goldset 快照釘定）

- **A3 權威度(in-degree 飽和加分):試過、量過、殺了。** 雙面負增益——search train nDCG 0.875→0.862(w=0.1)→0.847(w=0.2);edit P@8 0.700→0.653(w=0.1)。**負結果的解釋**:本圖高連結節點=導覽樞紐(MOC/總覽型),不是答案本體——與盲標「MOC 關鍵字磁鐵」實證同構;TREC「連結證據幫導航型查詢」的前提是目標即樞紐,而 lumos 查詢的目標是葉子內容節點,方向恰好相反。code 已刪(留墓碑註防憑直覺復活);調研表的「低優先(shadow 驗證)」裁定走完閉環。
- **A1.5 狀態降權(stale/superseded ×mult):本庫測不出(零效果——快照語料 stale 樣本不足),旋鈕保留、預設 1.0=關**(`LUMOS_RANK_STATUS_MULT`)。適用時機=stale 樣本多的庫(如 Landmark 152 筆 Verification 史);啟用前先在該庫考卷 A/B。
- **A2 事件 frecency:事件帳已種**(見 [[Systems/retrieval-ranking]]),等語料累積後照 Firefox 教訓(查詢時現算/半衰期長/係數少)做。
