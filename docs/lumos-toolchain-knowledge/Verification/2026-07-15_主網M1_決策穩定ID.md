# 驗證：關係層主網 M1——決策穩定 ID

主網四里程碑的第一座（[[關係層主網_實作計畫]] M1，spec v4 經 3 輪 design-loop 人裁實質收斂後開建）。

## 變更範圍（scripts/lumos）
- **`cmd_decision_add`**：指派 `id: d<max+1>`（`_max_decision_id` 掃既有取 max，不回收缺號）；寫後自驗升級 **ID 精確驗證**；回 `(0, "<rel>#d<N>")`。
- **`cmd_decision_supersede`**：**選擇端唯一命中**（子字串多重命中 → rc=2 列候選，殺掉「先改錯項再用錯項 id 驗證它改成功」的假安心）+ **`#dN` 精確定址**（未 reindex 查無 → rc=2 提示）；讀 target id 組全域格式回傳 `(0, gid)`；有 id 用 ID 精確自驗、無 id（未 reindex 舊節點）退回 content 驗證（相容）。
- **`cmd_decision_reindex <節點>`**（新指令）：無 id 決策依文件順序從 max+1 接續編號（由下而上插入防位移）、冪等、混合狀態絕不撞既有 id、解析不到明確拒絕；自驗＝全數有 id ∧ 唯一 ∧ 既有 id 未動。
- **dispatcher**：解包 `(rc, decision_id)`、CLI 對外仍回 int（M4 [S1] 觸發端將在此接 cascade_surface）。
- **doctor Check E3**（新，E2 之後）：`decision_refs` 指向的決策 `valid:false` → 斷義 warn；格式壞/節點不存在/id 未指派 → dangling warn（不靜默，對齊反靜默紀律）。
- **doctor Check E2 精化**：鄰居有 `decision_refs` → 只在指到「那條」翻案決策才標（指別條不冤枉）；無 refs → 退回節點+ended 級。
- `scripts/test_lumos.py`：7 個新測試函式、26 斷言。
